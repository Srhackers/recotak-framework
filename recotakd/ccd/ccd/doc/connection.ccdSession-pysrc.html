 30  how incoming requests are handled 
 31  ---------------------------------- 
 32   
 33  there is a three staged verification. the first stage verifies that a valid 
 34  session id is provided. without session id it is allowed to login in order to 
 35  get one. the second stage checks for a valid request id. without request id it 
 36  is allowed to request one or to write to plugins stdin, which is relevant for 
 37  interactive plugins. the third stage verifies, that the user has selected a 
 38  project. without selecting a project, it is only allowed to create or manipulate 
 39  projects. executing plugins for instance remains forbidden.
